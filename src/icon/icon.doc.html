Icon

Shows an icon with different sizes

How to configure it

Sizes

Update type IconSize in icon/api.ts with your own list of sizes. Then update the mapping sizes in icon/icon.ts.

Icon format

Sometimes, you can have several formats for your icon (white / black, primary / secondary, …). You can either declare the variants as a different type (ex: <icon type="plus-white">) or as a type with a variant (ex: <icon type="plus" variant="white">).

Declaration of icons

This project uses a simple javascript function returning a svg element (with lit-html). However, it is not an obligation and will depends on several causes (tooling, technical format of the icon, …).

Internationalization

If you want to use or update the component, think about:

  • Colors and forms: The meaning per country and region
  • Give a translated title and a description to your icon
  • Avoid using icons without another text to describe them (or make the context explicit)

Accessibility

If you want to use or update the component, think about:

  • Clear label and description for your icon (title and description in svg, alt text for image)
  • Colors and contrast of icons and their usage in other components (ex: buttons)
  • Size of the icon
  • Having icon different enough to avoid confusion

Best practices

Do

Prefer SVG

Prefer SVG over other formats. You can give information like a title and a description readable by screen readers. They can be stylized with css without be forced to create another file. Also, they often end up having a lower size.

Don't

Replace all texts with icons

Icons give graphical information and help with people with dyslexia. However, they also remove clear information that a text can bring. Prefer having both (icon and text) instead of only icons.
